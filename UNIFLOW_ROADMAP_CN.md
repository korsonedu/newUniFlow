## 0. 目标定义（商用完成标准）
当且仅当以下四项全部达成，视为“可商用发布”：
1. 产品可用：教师可在 10 分钟内完成“导入课件 -> 录课 -> 时间轴编辑 -> 导出”的闭环。
2. 工程可交付：macOS 原生包可重复构建、安装、回归、发布。
3. 结果可依赖：预览与导出一致，长时录制稳定，无关键功能阻断。
4. 合规可落地：满足中国教育软件上线所需的基础合规与运营材料。

## 2. 中国市场定位与独创性（商用策略）
> 结论（基于主流教育工具公开能力的推导）：国内常见产品强于“课件资源/直播教学”，弱于“录后时间轴级精修 + 音画动作一体回放一致性”。

UniFlow 的差异化必须固定为：
1. **录后可编辑时间轴**：不是只录屏，而是可插删/重排/ripple/split/retime 的“可编排课程资产”。  
2. **白板-音频-动作强绑定**：任意结构编辑后，笔迹、波形、动作、导出保持同一时基。  
3. **所见即所得导出**：预览即成片，减少老师反复试导出。  
4. **macOS 原生优先稳定**：面向高频录课老师，强调低延迟与可预期。

---

## 3. 执行总则（强制）
1. 时间轴单一真值：禁止绕过事件直接改状态。  
2. 时间轴可编辑能力必须可回放、可撤销、可重做。  
3. 白板书写只允许 `perfect-freehand` 一条运行管道。  
4. 平台顺序固定：`macOS -> iPadOS -> Windows`。  
5. 每轮必须执行并记录：
   - `npm run typecheck`
   - `npm run test:timeline`
   - `npm run build`
6. 涉及发布闭环轮次必须额外执行：
   - `npm run verify:macos`
7. 路线执行顺序固定为：
   - 先 `C线（代码稳定性全面性）`
   - 后 `P线（UI + 功能完善）`
   - 最后 `B线（商用清单）`

## 4. 三线总路线（代码层/产品层/业务层）
执行顺序：`C -> P -> B`

## C 线：代码层（可验证、可维护、可发布）
### C1（P0）时间轴与渲染确定性
交付项：
1. `native vs TS` 一致性回归扩展到：`insert/delete/delete-range/ripple/split/retime` 全覆盖。  
2. 波形可见性硬断言：`path数量>0 && 图层可见 && 高度>阈值`。  
3. 导出一致性指纹：预览帧与导出帧关键时间点一致。

验收门槛：
1. 一致性回归 100% 通过。  
2. 波形“有数据但看不见”问题在自动化中可复现、可阻断。  

### C2（P0）白板输入与性能稳定
交付项：
1. 书写链路仅 `pointer/coalesced/raw -> perfect-freehand -> path`。  
2. 录制态写字流畅性压测（5/15/30 分钟）。  
3. 交互性能观测：输入点数、渲染耗时、掉帧率统计。

验收门槛：
1. 30 分钟录制阶段无“写一笔卡死下一笔”阻断。  
2. 关键机型下书写与抬笔固化无明显跳变。  

### C3（P1）发布工程化
交付项：
1. macOS 构建脚本、发布清单、回滚清单统一。  
2. 构建产物可复现（同版本可重复打包）。  
3. 发布记录模板落地（版本、验证、阻塞、修复）。

验收门槛：
1. 连续 3 次发布演练均通过核心门禁。  

---

## P 线：产品层（老师可用、好用、爱用）
### P1（P0）一屏录课闭环
交付项：
1. 白板全窗口可写（含顶部边界、越界连续书写策略）。  
2. 顶栏/工具栏/时间轴悬浮化，不侵占核心书写区。  
3. 录制中自动收拢低频控件，停止后恢复。

验收门槛：
1. 1366x768 ~ 2560x1600 均可完成连续录课不遮挡。  

### P2（P0）录后精修效率
交付项：
1. 时间轴操作区与动作区与预览区职责清晰。  
2. 波形与页面预览同区展示，编辑语义一致。  
3. 常用操作（剪切/重排/删除未来）可在 2 次点击内完成。

验收门槛：
1. 典型老师任务（删口误、补讲解、换页）成功率 >= 95%。  

### P3（P1）成片质量与可控导出
交付项：
1. 导出预设（课堂高清/低带宽）  
2. 导出失败可恢复（明确错误+可执行动作）  
3. 导出过程可继续编辑（后台队列）

验收门槛：
1. 导出成功率 >= 98%（试点样本集）。

---

## B 线：业务层（仅保留 B1，直达商用）
### B1（P0）macOS 商用闭环（唯一业务路线）
交付项：
1. 试点版本（内测）  
- 10-30 位老师连续使用 2 周，采集故障与留存。  
2. 商业能力最小集  
- 激活码/许可证、版本管理、崩溃上报、更新策略。  
3. 合规与交付材料  
- 用户协议、隐私政策、数据处理说明、售后 SLA、发票/合同模板。  
4. 销售可讲清  
- 一句话价值、三大场景、与竞品差异演示脚本。

验收门槛：
1. 试点老师周留存 >= 70%。  
2. 关键任务 NPS >= 30。  
3. 完成首批付费交付能力（可签约、可交付、可售后）。

## 5. 里程碑（按顺序执行）
## M0-M1（第 0-2 周）：代码稳定性全面收口（仅 C 线）
1. 完成稳定性矩阵：输入、时间轴编辑、导出、异常恢复全覆盖。  
2. 完成 Native/TS 一致性矩阵：短项目 + 长项目 + 批量编辑。  
3. 完成 30 分钟录制/编辑/导出压力回归并固化自动化门禁。

## M2（第 3-4 周）：UI 与功能完善（仅 P 线）
1. 一屏录课体验统一：布局、尺寸、层级、交互反馈全部精修。  
2. 时间轴操作效率优化：常用任务最短路径、操作可预期、误触可恢复。  
3. 导出体验完善：预设、错误恢复、结果提示统一。

## M3（第 5-6 周）：商用清单落地（仅 B 线）
1. 试点商用材料：激活/协议/隐私/售后/版本策略。  
2. 发布演练与回滚机制闭环。  
3. 达成“商用完成标准”四项全绿后再对外发布。

## 6. 风险清单与提前处置
1. 书写链路再次分叉风险  
- 处置：CI 增加“禁止非 perfect-freehand 运行路径”检查。

3. 原生适配回退风险  
- 处置：native/TS 双引擎对比回归作为发布门禁。  
4. 合规滞后风险  
- 处置：M1 即启动法务文档，不等发布末期。

## 7. 商用出货门禁（Release Gate）
全部满足才允许对外商用：
1. 功能门禁  
- 时间轴核心命令、白板书写、波形显示、导出全通过。  
2. 质量门禁  
- `typecheck/test:timeline/build/verify:macos` 全通过。  
3. 性能门禁  
- 30 分钟连续录制无关键卡死，CPU/内存在目标机型可接受。  
4. 合规门禁  
- 协议、隐私、数据处理、售后机制齐备。  
5. 运营门禁  
- 可激活、可升级、可回滚、可支持。
